## 本書の目的

本書は JavaScript エンジン V8 の内部実装を、ソースコード (`/home/user/v8` を一次資料) の行番号レベルまで踏み込んで解剖する技術書です。Tagged Pointer の最下位ビット 1 つに込められた設計判断から、TurboFan の Sea of Nodes による最適化、Orinoco GC の三色マーキング、Torque DSL から CodeStubAssembler を経て embedded blob に焼き込まれるビルトイン、`Array.prototype.flat` の TryFastFlat 二パス実装に至るまで、エンジン全体を一冊で見渡せる構成にしました。

V8 の解説資料は断片的なものが多く、ある資料は Hidden Class を、別の資料は GC を、また別の資料は JIT を扱う構造になりがちです。しかし V8 の各レイヤは深く絡み合っており、Hidden Class を理解せずに Inline Cache を、Tagged Pointer を理解せずに GC を、ElementsKind を理解せずに `Array.prototype.flat` を語ることはできません。本書では、ひとつの語彙体系で 50 章を貫き、各章がそれ以前の章で導入された概念に依拠して積み上がっていく構成を目指しました。

## 各部の役割

### 第 I 部 V8 基礎 (ch02 - ch13)

V8 を理解するための土台となる 12 章です。Tagged Pointer の最下位ビット (`kHeapObjectTag = 1`, `kSmiTag = 0`) の意味づけから始まり、HeapObject の先頭バイトが必ず Map ポインタである不変条件、Map (Hidden Class) の Transition Tree、JSObject の in-object / out-of-object プロパティ、ElementsKind の 21 種類の状態、String の 5 階層 (Seq / Cons / Sliced / Thin / External)、HeapNumber と BigInt、Pointer Compression Cage と V8 Sandbox、Heap Spaces (New / Old / Code / RO / LO)、Orinoco GC (Scavenger / Mark-Compact)、実行パイプライン (Ignition / Sparkplug / Maglev / TurboFan)、Inline Cache と FeedbackVector までを縦串で解剖します。

### 第 II 部 メモリ管理深掘り (ch14 - ch18)

第 I 部で導入した概念を、メモリ管理の縦軸で再俯瞰します。第 I 部が「V8 全体を 12 章で 1 周する」のに対し、第 II 部は「メモリ管理だけを 5 部で 1 周する」構成です。同じ Tagged Pointer や Map を扱っていても、参照ファイルや視点が異なるため、第 I 部の理解を別の角度から強化できます。重要数値早見表 (Tagged Pointer のタグ値、Page サイズ、Sandbox サイズ、Smi 範囲、`kMaxFastArrayLength` など) は ch14 の冒頭にまとめてあるので、用語集として活用してください。

### 第 III 部 配列の深層 (ch19 - ch35)

JavaScript の `Array` に焦点を絞り、ElementsKind の 21 種類 (PACKED_SMI_ELEMENTS から SHARED_ARRAY_ELEMENTS まで) の物理表現と遷移ラティス、push / pop / shift / unshift / slice / concat / map / filter / flat / sort の内部実装、Protector cells (NoElementsProtector など) を使った楽観的最適化、AllocationSite と Pretenuring、TypedArray の RAB / GSAB、`indexOf` の SIMD 実装まで踏み込みます。第 I 部の ch06 (ElementsKind) を読んだ前提で、より深く展開します。

### 第 IV 部 Torque (ch36 - ch49)

V8 のビルトイン記述に使われる DSL である Torque について、Earley パーサーから始まる言語処理系、CFG への変換、CodeStubAssembler / TurboShaft Assembler への接続、`BUILD.gn` を経て embedded blob に焼き込まれるまでの工程を扱います。`Array.prototype.map` や `Promise.prototype.then` などの代表的ビルトインの実装パターンも紹介します。第 III 部 ch24 (Torque ― V8 独自のビルトイン言語) で概略を掴んでから読むと、より深く理解できます。

### 第 V 部 ケーススタディ (ch50)

`Array.prototype.flat` の実装を、Torque ソース `src/builtins/array-flat.tq` 全 610 行を起点に、TryFastFlat の二パス設計、FastJSArrayWitness パターン、Bailout の分類、ECMAScript 仕様との対応、コミット史までを通しで読み解きます。第 I 部 〜 第 IV 部で導入した語彙 (Tagged Pointer、ElementsKind、Witness、Torque) が、ひとつの builtin にどう組み合わさるかを観察できます。

## 前提知識

JavaScript の言語仕様 (ECMAScript) の基本構文、関数、プロトタイプチェーン、`Array`、`TypedArray`、`Promise` の利用経験を前提とします。C++ のテンプレート、ポインタ、メモリレイアウト、アライメントの基礎知識があると、HeapObject やテンプレートクラスの説明が読みやすくなります。CPU アーキテクチャ (x86_64 / ARM64) のレジスタ、キャッシュ、命令セットに親しんでいると、Tagged Pointer の符号拡張トリックや SIMD 最適化の節が腑に落ちます。

## 引用と出典について

本書のコード引用はすべて V8 main ブランチを一次資料とし、`ファイルパス:行番号` の形式で出典を併記しています。引用箇所はそのまま `cd /path/to/v8 && less -N src/objects/tagged.h` のように手元で確認できます。記述の対象とした V8 のリビジョンは執筆時点の main HEAD であり、リポジトリの将来のコミットで行番号がずれる場合があります。

## V8 全体俯瞰図

│  │ ┌────────────────────────┐ │  │  (複数 Isolate で共有)    │  │
│  │ │ NewSpace (Scavenger)   │ │  │  Map、定数、builtin Code  │  │
│  │ │ OldSpace (Mark-Compact)│ │  └───────────────────────────┘  │
│  │ │ CodeSpace (executable) │ │                                 │
│  │ │ TrustedSpace (Sandbox 外)│                                │
│  │ │ LargeObjectSpace       │ │                                 │
│  │ └────────────────────────┘ │                                 │
│  └────────────────────────────┘                                 │
│                                                                 │
│  ┌─────────────────────────────────────────────────────────┐    │
